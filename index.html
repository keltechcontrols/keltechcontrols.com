Scheduler Development

Not software training.
Field-ready scheduling.

The goal is not to make someone comfortable clicking around P6. The goal is to build real scheduling capability—how to structure work, build logic that flows, update with discipline, and make better decisions when reality starts pulling the schedule apart.

Since clients with their own P6 access can open example files directly, you can use .XER schedules as working assignments between sessions—not just static screenshots or theory.

Build

How to structure schedules that flow from start to finish, avoid unnecessary hard ties, limit constraints, and maintain usable flexibility for turnarounds.

Execute

What happens when the shutdown starts, pre-TA work is not where it should be, contingency gets tested, and the schedule has to become a control system—not just a plan.

Control

How to read negative float, watch near-critical work, update with discipline, separate build and update layouts, and use milestones and baseline integrity to manage progress.
